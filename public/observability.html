Caching Architecture Optimizations

We run a high-fidelity **Cache-Aside Pattern** powered by **Valkey 8** to protect relational database performance. This keeps API response times under **1.14s** and ensures we avoid redundant, expensive table queries.

Cache Read Patterns Technicians and Properties lists are cached with a 5/10 min TTL. Reads hit memory instead of Postgres.
Cache Invalidation Work Order writes (`POST`, `PUT`, `DELETE`) immediately trigger scan-based cache invalidation.
